Integrating different models, data and computers to solve real world problems.
Model Repository

Collecting model resources to build a dictionary where all models (also include related tools, algorithms, ect.) be organized in a formal way. Users can discover a model to find its detailed descriptive information, conceptual description, logical description, computable resources, evolution history, application scenarios and so forth. We only publish users' resources under their clear permission.

Data Repository

Collecting data resources to build a harbor where users can explore modeling-related data through an universal center. Data resources can be from the sharing of users with their public option. And various data sharing sites are linked to help users search interested data without go here and there. We only publish users' resources under their clear permission.

Using Models as Services

Model as a Service, Data as a Service and Computer as a Service in an open web environment. Users can setup input data and run a model via a web client, and the related model would be executed in a remote computer node. A user can invoke a model service before boarding and obtain results when get off. And a set of solutions are offered to help users convert original models as reusable services, publish data files as resuable services and share computers as available services.

Thematic Center

Regarding a research topic or a specific problem, multi-disciplinary knowledge could be involved. Several thematic centers are constructed to help researchers collect models, data and other related resources. Topic-related or problem-related resources could be easily discovered within a thematic center. And various web applications would be offered in certain centers.

Participatory Modeling Platform

To solve a complex problem, single human's expertise may be not enough. Collaboration is important. Participants from different areas can contribute resources and ideas which would largely benefit the problem-solving. With the Participatory Modeling Platform, dispersal participants can build a team, construct a project, upload various resources, communicate with other, propose solutions, building new models and take other responsibilities which can help solving the target problem.

Supportive Tools

Model Service container

Model Service Container is developed based on the open source NodeJS, and it mainly provides tools/functions for users to deploy a model as a reusable model-service which can also be managed in the model service container. It can be deployed in various computing platforms (e.g. Windows, Linux, MaxOSX, etc.).

Use Now
Data Service Container

Data Service Container is developed based on the open source NodeJS project, and it mainly provides tools/functions for users to publish data as reusable data-services. It can be deployed in various computing platforms (e.g. Windows, Linux, MaxOSX, etc.).

Use Now
Model Wrapping Tools

Model Wrapping Tools are some tools that help users to wrap their own models as services. Tools include Programming Libraries for converting original program as service-enabled program (currently supporting C++, C#, Java and Python) and Model Service Packaging tool for packaging model execution related files as one zip file.
